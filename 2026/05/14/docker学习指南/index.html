一、Docker 概述与核心概念

1.1 什么是 Docker？

Docker 是一个开源的容器化平台，允许开发者将应用程序及其依赖打包成标准化的容器，在任何环境中一致运行。

传统部署问题：

  • “在我机器上能跑” - 环境不一致
  • 依赖冲突 - 不同应用需要不同版本
  • 部署复杂 - 手动配置繁琐

Docker 解决方案：

  • 环境一致性：一次构建，到处运行
  • 资源隔离：容器间相互隔离
  • 快速部署：秒级启动，轻量级
  • 版本控制：镜像可版本化管理

1.2 核心概念

概念 说明 类比
镜像 (Image) 只读模板，包含运行应用所需的所有内容 操作系统 ISO 镜像
容器 (Container) 镜像的运行实例 虚拟机实例
仓库 (Registry) 存储和分发镜像的服务 应用商店
Dockerfile 定义如何构建镜像的文本文件 配方
卷 (Volume) 持久化存储数据 外接硬盘
网络 (Network) 容器间通信机制 虚拟网络

1.3 Docker 架构

关键组件：

  • Namespaces：提供进程隔离（PID, NET, MNT, UTS, IPC）
  • Cgroups：限制资源使用（CPU, 内存, 磁盘 I/O）
  • UnionFS：分层文件系统（AUFS, OverlayFS）
  • SELinux/AppArmor：安全策略

1.4 Docker 与虚拟机对比

特性 Docker 容器 虚拟机
启动时间 秒级 分钟级
资源占用 轻量（共享内核） 重量（完整 OS）
隔离性 进程级 硬件级
性能 接近原生 有虚拟化开销
镜像大小 MB 级别 GB 级别
适用场景 微服务、CI/CD 完全隔离环境

4.1 镜像分层原理

Docker 镜像采用分层存储结构，每层都是只读的：

优势：

  • 节省空间：多个镜像共享相同层
  • 快速构建：利用缓存，只构建变化的层
  • 版本控制：每层都有独立的哈希值

一、环境准备与基础配置

1.1 安装与启动

  • 下载地址：https://jmeter.apache.org/download_jmeter.cgi
  • 依赖：JDK 8 或更高版本
  • 启动方式：
    • Windows：bin/jmeter.bat
    • Linux/Mac：bin/jmeter.sh

1.2 推荐初始设置

设置项 路径 建议值
界面语言 Options → Choose Language → Chinese (Simplified) 中文
字体大小 Options → Zoom In 120% ~ 150%
内存调整 编辑 bin/jmeter.bat 或 jmeter.sh set HEAP=-Xms2g -Xmx4g

HTTP 请求默认值 (HTTP Request Defaults)

参数 示例值
Protocol http / https
Server Name 192.168.234.128
Port Number 8000
Connect Timeout 30000
Response Timeout 30000

作用：后续 HTTP 请求只需填写 Path 和 Method，无需重复配置域名端口。

HTTP 信息头管理器 (HTTP Header Manager)

名称 值
Content-Type application/json; charset=UTF-8
Accept application/json
Authorization Bearer ${token}

可挂在线程组下（全局生效），也可挂在单个请求下（仅该请求生效，优先级更高）。

Stop Thread on EOF false
Sharing Mode All threads

场景 JSON Path Additionally assert value Expected value Match as regex

4.2 性能测试必做调整

调整项 操作 原因
禁用查看结果树 右键 → Disable 极耗内存，大并发下会导致 JMeter 崩溃
聚合报告保存到文件 Filename 填写 ./result.jtl 持久化结果，后续分析
减少监听器数量 只保留聚合报告 降低资源消耗
使用非 GUI 模式 jmeter -n -t xxx.jmx GUI 模式消耗大量系统资源
调整 JVM 内存 set HEAP=-Xms2g -Xmx4g 防止内存溢出

添加路径：右键 线程组 → Add → Sampler → JDBC Request

5.4 Query Type 选择指南

Query Type 是否支持 ? 使用场景
Select Statement ❌ 不支持 SQL 中无变量，直接写完整语句
Prepared Select Statement ✅ 支持 SQL 中使用 ? 占位符传参（推荐，防注入）
Update Statement ❌ 不支持 INSERT/UPDATE/DELETE，无参数
Prepared Update Statement ✅ 支持 INSERT/UPDATE/DELETE，带 ? 参数
Callable Statement — 调用存储过程

七、常见问题与解决方案

7.1 导入 .jmx 文件异常

现象：导入时报错或无法正常显示。

解决：

  1. 确认 JMeter 版本与文件版本兼容（高版本可打开低版本）
  2. 手动从零搭建测试计划，逐步添加元件
  3. 保存时确保文件编码为 UTF-8

7.2 JDBC 报错：Unknown column ‘xxx’ in ‘where clause’

现象：SQL 执行报错，提示列不存在。

原因：字符串值未加单引号，MySQL 将值解析为列名。

解决：

或使用 Prepared Statement（参数值不加引号）：

7.3 JDBC 报错：You have an error in your SQL syntax near ‘?’

现象：使用 ? 占位符时报语法错误。

原因：Query Type 选成了 Select Statement，不支持 ?。

解决：改为 Prepared Select Statement 或 Prepared Update Statement。

7.4 JDBC 查询只显示表头，无数据行

排查步骤：

  1. 确认参数值是否正确传入（添加 Debug Sampler 查看变量）
  2. 用固定值直接测试（排除变量问题）
  3. 用 MySQL 客户端执行相同 SQL，确认数据库有数据
  4. 检查 JDBC URL 中的数据库名是否正确

7.5 变量取不到值（显示 ${xxx}）

原因：

  • 变量名拼写错误
  • 变量定义位置不对（全局变量 vs 线程组变量）
  • CSV 文件未正确读取（路径错误、编码问题）

解决：

  • 使用 Debug Sampler 打印所有变量值
  • 在 JSR223 脚本中打印：log.info(vars.get("xxx"))
  • 检查 CSV 配置：文件名、路径、编码、变量名

7.6 连接超时 / 拒绝连接

排查：

  1. 浏览器直接访问接口地址，确认服务是否启动
  2. ping 服务器 IP，确认网络可达
  3. 检查防火墙是否放行对应端口
  4. 检查服务是否绑定 127.0.0.1（只能本机访问）

7.7 中文乱码

解决：

  • JDBC URL 加 useUnicode=true&characterEncoding=UTF-8
  • HTTP 信息头管理器加 Content-Type: application/json; charset=UTF-8
  • JMeter 全局编码：File → Encoding → UTF-8

八、非 GUI 模式运行

8.1 为什么用非 GUI 模式

GUI 模式会消耗大量系统资源（内存、CPU），不适合正式压测。官方推荐：

  • 调试阶段：GUI 模式
  • 正式压测：非 GUI 模式

8.2 运行前准备

  1. 保存 .jmx 文件
  2. 禁用查看结果树（必须）
  3. 聚合报告填写 Filename（如 ./result.jtl）
  4. 关闭 JMeter GUI

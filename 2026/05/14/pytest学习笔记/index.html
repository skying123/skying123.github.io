Pytest 完整学习笔记

基于实战项目总结，涵盖从入门到进阶的全部核心知识点。

一、项目结构

二、配置文件 pytest.ini

核心配置项

配置项 作用
testpaths 定义 pytest 从哪些目录开始找测试
python_files 定义什么样的文件会被当成测试文件
addopts 每次运行 pytest 自动附加的参数
markers 注册自定义标记,避免 PytestUnknownMarkWarning

常用内置标记

标记 作用
@pytest.mark.skip(reason="...") 无条件跳过
@pytest.mark.skipif(condition, reason="...") 条件满足时跳过
@pytest.mark.xfail(reason="...") 预期失败,失败不计入错误
@pytest.mark.xfail(strict=True, ...) 预期失败,但如果通过了反而报错
@pytest.mark.usefixtures("name") 自动使用指定 fixture

calculator.py

user_service.py

四、Fixture 机制(conftest.py)

conftest.py 是 pytest 的灵魂: 文件名固定,自动发现,无需 import,放在哪个目录,该目录及子目录的测试都能使用其中的 fixture。

Scope 对比

scope 生命周期 创建次数 适用场景
function 每个测试函数 N 次 默认,数据隔离最彻底
class 每个测试类 类数次 类内共享
module 每个测试文件 文件数次 模块级资源
package 每个包目录 包数次 较少用
session 整个 pytest 进程 1 次 全局资源

Fixture 的核心作用

  1. 提供测试数据
  2. 创建被测对象
  3. 环境准备与清理(yield)
  4. 控制资源生命周期(scope)
  5. 模拟/替换外部依赖
  6. 自动执行的前置操作(autouse)

五、基础测试(test_basic.py)

pytest vs unittest 写法对比

场景 unittest pytest
基本断言 self.assertEqual(a, b) assert a == b
异常断言 with self.assertRaises(...) with pytest.raises(...)
浮点比较 无原生支持 pytest.approx(...)
类要求 必须继承 TestCase 普通函数即可

六、Fixture 实战(test_fixture.py)

常用内置 fixture

fixture 作用
capsys 捕获标准输出/错误
tmp_path 提供自动清理的临时目录(pathlib.Path)
tmpdir 旧版临时目录(py.path.local)
monkeypatch 临时修改环境、函数、字典
caplog 捕获日志输出
request 获取当前测试的元信息

七、参数化测试(test_parametrize.py)

参数化核心要点

特性 说明
pytest.param(..., id="...") 自定义用例显示名称
pytest.param(..., marks=...) 给单组数据加标记
多个装饰器叠加 自动做笛卡尔积
indirect=["name"] 参数先传给 fixture 预处理
装饰器贴在类上 类内所有方法共享参数

八、进阶特性(test_advanced.py)

monkeypatch 常用方法

方法 作用
setenv(key, value) 临时设置环境变量
delenv(key) 临时删除环境变量
setattr(obj, name, value) 临时替换对象属性/函数
delattr(obj, name) 临时删除属性
setitem(dict, key, value) 临时修改字典项
delitem(dict, key) 临时删除字典项

关键原则: 所有修改都是测试级临时的,结束后自动恢复,互不干扰。

九、命令行速查表

十、pytest vs unittest 核心对比

维度 unittest pytest
测试类 必须继承 TestCase 不需要继承任何类
断言 self.assertEqual() 原生 assert
异常断言 self.assertRaises() pytest.raises()
浮点比较 无原生支持 pytest.approx()
参数化 subTest 繁琐 @pytest.mark.parametrize
Fixture setUp/tearDown 固定 @pytest.fixture 灵活可组装
标记 无 @pytest.mark.xxx 丰富
插件生态 几乎没有 极其丰富
发现机制 需指定 自动递归发现

十一、学习路径建议

核心口诀: 函数即测试,assert 即断言; fixture 管后勤,parametrize 管数据; monkeypatch 管环境,scope 管生命周期。

5.4 显式 vs 隐式

特性 隐式等待 显式等待
作用范围 全局 局部
等待条件 固定（元素存在） 可自定义多种条件
灵活性 低 高
性能 全局拖慢 精准等待
推荐度 低 高

不要混用两者，容易混乱。生产环境只用显式等待。

六、预期条件（EC）详解

6.1 元素存在与可见

条件 说明
presence_of_element_located 元素存在于 DOM（不一定可见）
presence_of_all_elements_located 所有匹配元素存在于 DOM
visibility_of_element_located 元素可见（存在+宽高>0+不隐藏）
visibility_of 传入元素对象判断可见性
invisibility_of_element_located 元素不可见或不存在

6.2 元素可交互

条件 说明
element_to_be_clickable 元素可见且启用
element_to_be_selected 元素被选中
element_located_to_be_selected 定位后判断是否选中

6.3 文本与内容

条件 说明
text_to_be_present_in_element 元素文本包含指定内容
text_to_be_present_in_element_value 输入框 value 包含指定内容

6.4 页面状态

条件 说明
title_contains 标题包含某文本
title_is 标题完全等于
url_contains URL 包含某字符串
url_to_be URL 完全等于
url_changes URL 发生变化

6.5 Frame 与 Alert

条件 说明
frame_to_be_available_and_switch_to_it iframe 可切换并自动切换
alert_is_present 弹窗出现

6.6 元素状态变化

条件 说明
staleness_of 元素从 DOM 消失
element_attribute_to_include 元素包含某属性

十四、异常处理与调试技巧

14.1 常见异常

异常 原因 解决
NoSuchElementException 元素未找到 检查定位器；使用显式等待
TimeoutException 等待超时 增加超时时间；检查条件
StaleElementReferenceException 页面刷新后元素引用失效 重新查找元素
ElementNotInteractableException 元素被遮挡或未加载 等待元素可点击；滚动到可见
ElementNotVisibleException 元素不可见 检查 CSS 隐藏属性
InvalidSelectorException 选择器语法错误 检查 XPath/CSS 语法
SessionNotCreatedException 驱动与浏览器版本不匹配 更新 ChromeDriver
NoAlertPresentException 弹窗未出现 先 wait.until(EC.alert_is_present())
UnexpectedAlertPresentException 弹窗突然出现阻塞操作 提前处理或捕获异常

Q1：find_element_by_id 报错不存在？

Q2：ChromeDriver 下载慢？

使用国内镜像或手动指定路径：

Q3：遇到验证码怎么办？

  • 添加反检测配置降低触发概率
  • 使用 Cookie 保持登录状态
  • 对接打码平台（如超级鹰）
  • 换测试站点练习（如 bing）

Q4：close() 关错了窗口？

close() 关闭的是当前焦点窗口。打开新窗口后必须先 switch_to.window() 切换焦点。

Q5：元素明明存在却点不到？

  • 元素在 iframe 里 -> 先 switch_to.frame()
  • 元素被遮挡 -> 滚动到可见或等待遮罩消失
  • 元素还没加载完 -> 使用显式等待

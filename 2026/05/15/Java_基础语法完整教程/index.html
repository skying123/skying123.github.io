编码建议

  1. 命名规范：类名大驼峰，方法/变量小驼峰，常量全大写
  2. 注释习惯：类和方法写文档注释（/** */），关键逻辑写行注释
  3. 异常处理：不要捕获了异常却什么都不做，至少记录日志
  4. 资源管理：使用 try-with-resources 自动关闭资源
  5. 空值检查：使用对象前先做 null 检查，或使用 Optional
  6. 字符串拼接：循环中大量拼接使用 StringBuilder，不要直接用 +
  7. 集合选择：频繁查询用 ArrayList，频繁增删用 LinkedList，去重用 Set
  8. equals 和 hashCode：自定义类用作 Map 键时，必须同时重写两者
  9. 避免魔法数字：用有意义的常量代替硬编码的数字
  10. 代码复用：提取公共方法，遵循 DRY 原则（Don’t Repeat Yourself）

学习路径建议：

  1. 先掌握基础语法（变量、运算符、流程控制）
  2. 理解面向对象（类、对象、封装、继承、多态）
  3. 熟悉常用类（String、集合、IO）
  4. 学习异常处理和泛型
  5. 进阶：多线程、Lambda、Stream API
  6. 实践：写小项目巩固知识

本文档涵盖 Java SE 基础语法，适合初学者系统学习。建议配合实际编码练习，加深理解。
